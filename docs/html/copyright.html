SQLite Is Public Domain


SQLite is in the
Public Domain

SQLite のすべてのコードとドキュメントは、 作成者によって public domain に捧げられています。 すべてのコード作成者、および彼らが勤務する企業の代表者は、 パブリック・メインへの貢献に献身する宣誓供述書に署名しており、 これらの署名済み宣誓供述書の原本は Hwaci 本社の防火金庫に保管されています。 すべての寄稿者は、 創造的な作品をパブリック・ドメインに捧げることを許可する国の市民です。 ソース・コード形式またはコンパイルされたバイナリとして、 目的または商用または非商用または手段を問わず、 誰でもオリジナルの SQLite コードを自由にコピーや変更や公開や使用やコンパイルや販売や配布することができます。

上記段落は、 SQLite の成果物コードとドキュメントに適用されます。 これらは SQLite ライブラリのパーツであり、 実際にバンドルしてより大きなアプリケーションに同梱します。 ビルド・プロセスの一部として使用される一部のスクリプト (たとえば、autoconf によって生成される "configure" スクリプト) は、 他のオープン・ソース・ライセンスに該当する場合があります。 ただし、 これらのビルド・スクリプトから最終的な成果物 SQLite ライブラリに到達するものはありません。 そのため、 これらのスクリプトに関連付けられたライセンスは、 SQLite ライブラリをコピーして使用する権利を評価する際の要素にはなりません。

SQLite で提供されるコードはすべてゼロから作成されています(scratch)。 他のプロジェクトや、 オープンなインターネットからコードを取得していません。 コードのすべての行は元の作者までさかのぼることができ、 これらの作者はすべて、 パブリックドメインへの献辞が登録されています。 そのため、SQLite のコード・ベースはクリーンで、 他のプロジェクトのライセンス・コードで汚染されていません。

Open-Source, not Open-Contribution

SQLite はオープンソースです。 つまり、SQLite のコピーを好きなだけ作成し、 それらのコピーを使って好きなことを無制限に行うことができます。 しかし、 SQLite は オープン貢献(open-contribution)ではありません。 SQLite をパブリック・ドメインに保ち、 コードが独自のコンテンツやライセンスされたコンテンツで汚染されないようにするために、 プロジェクトは、 パブリック・ドメインへの貢献を捧げる宣誓供述書を提出していない人々からのパッチを受け入れません。

SQLite のコードはすべてオリジナルであり、 SQLite で使用するために特別に作成されています。 インターネット上の未知のソースからコピーされたコードはありません。

Warranty of Title

SQLite はパブリック・ドメインにあり、ライセンスは必要ありません。 それでも、 一部の組織は、SQLite を使用する権利の法的証拠を求めています。 これが発生する状況には以下のようなものがあります:

  • あなたの会社は、著作権侵害の申し立てに対する免責(indemnity)を望んでいます。
  • パブリック ドメインを認めない司法管轄区域(jurisdiction)で SQLite を使用しています。
  • あなたは、 作品をパブリック・ドメインに捧げる著作者の権利を認めていない司法管轄区域(jurisdiction)で SQLite を使用しています。
  • あなたは SQLite を使用および配布する法的権利があることの証拠として、 有形の法的文書を保持する必要があります。
  • あなたの法務部門から、ライセンスを購入する必要があると言われました。

上記の状況のいずれかに該当する場合は、SQLite のすべての開発者を雇用している Hwaci が SQLite の所有権の保証を販売します。 タイトルの保証は、 主張されている SQLite の作成者が真の作成者であること、 および作者が SQLite をパブリック・ドメインに捧げる法的権利を持っていること、 および Hwaci がそれらの主張に対する異議申し立てに対して積極的に弁護することを主張する法的文書です。 SQLite のタイトル保証の販売による収益はすべて、 SQLite の継続的な改善とサポートに資金を提供するために使用されます。

Contributed Code

SQLite を完全に無料にし、 著作権の妨げにならないようにするために、 プロジェクトはパッチを受け入れません。 あなたが変更を提案したい場合、 概念実証(proof-of-concept)としてパッチを含めていただければ幸いです。 ただし、 我々がパッチを最初から書き直したとしても、 気分を害しないでください。
